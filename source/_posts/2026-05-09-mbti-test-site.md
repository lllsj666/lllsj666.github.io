## 前言

某天突然冒出一个念头：想做一个测试网站从零到上线。

没有多复杂的动机，就是觉得这件事有意思，想做出来看看。现在回头看，从本地敲下第一行代码，到网站正式上线、朋友开始分享测试结果，整个过程比预想的顺利，也比预想的好玩。

这篇文章是一份完整的项目复盘，涵盖技术选型、开发流程、部署上线和后续迭代。希望能给想做类似项目的朋友一些参考。

## 项目概览

**网站地址：[mbti-flame-ten.vercel.app](https://mbti-flame-ten.vercel.app/)** 5月9日 这个网站目前只有开代理网络、加速器、VPN才能访问。

一个 MBTI 性格测试网站，结合荣格八维认知功能，通过 36 道题目、四个维度，帮助用户了解自己的人格类型，生成详细报告并提供相处分析。

## 技术选型 & 工具链

项目本身不复杂，用到的工具也不多。列出它们不是为了显得"专业"，而是说清楚每样东西解决了什么问题。

| 工具 | 用途 |
|------|------|
| **Next.js** | 网站框架，负责路由、页面渲染、打包构建 |
| **Tailwind CSS** | 样式方案，全程零 CSS 文件，全部用 utility class 搞定 |
| **VS Code** | 编辑器，配合 AI 插件高效写代码 |
| **Claude Code**（内置 DeepSeek-V4Pro） | AI 辅助开发：排版、生成组件、调整动画，节省了大量重复劳动 |
| **Git + GitHub** | 版本控制，每次改动独立提交，出问题随时回退 |
| **Vercel** | 部署平台，关联 GitHub 仓库，代码推送后自动上线 |

VS Code 插件只装了两个：

- **Claude Code for VS Code**：在编辑器内直接与 AI 对话、让它修改文件，无需来回切换窗口
- **Tailwind CSS IntelliSense**：写 class 时自动补全，悬浮预览效果。没有它，写 Tailwind 的效率会大打折扣

## 从零到上线：GitHub + Vercel 实操

如果你也想做类似的项目，这套流程可以直接复用：

```bash
# 第一步：本地创建 Next.js 项目
npx create-next-app@latest mbti-test
cd mbti-test

# 第二步：安装 Tailwind（Next.js 14 以上自带，旧版本需手动安装）

# 第三步：在 GitHub 新建仓库，关联本地代码
git init
git add .
git commit -m "init: mbti test project"
git remote add origin <你的仓库地址>
git push -u origin main
```

然后打开 Vercel：

1. 注册 / 登录 [vercel.com](https://vercel.com)
2. 点击 **Add New → Project**，选择你的 GitHub 仓库
3. Vercel 会自动识别 Next.js 项目，无需手动配置
4. 点击 Deploy，等待约半分钟，线上地址生成

日常开发循环就三条命令：

```bash
npm run dev          # 本地启动，边改边看效果
git add .
git commit -m "描述本次改动"
git push origin main # 推送后 Vercel 自动部署
```

> 如果你遇到了 `merge conflict`，恭喜，说明你已经进入真实开发状态了。通常是和 AI 同时改了同一个文件的同一行，Git 不知道该保留谁的。在 VS Code 里手动选择保留哪一边，再 commit 一次即可。

## 功能优化迭代记录
26.5.9更新
### 首页

进入网站后，首页卡片会随机展示一种 MBTI 人格类型，并同步呈现对应的主题色——紫人紫色、蓝人蓝色，以此类推。每次刷新页面，人格类型和按钮颜色都会随机切换，让首页更具变化和趣味性。
### 答题页

- **进度条**：每答完一题，进度条向前推进；返回上一题时同步回退，答题进度一目了然。
- **进度缓存**：答题过程中不小心刷新页面，当前的题号 `ci` 也会存入 `localStorage`，刷新后自动恢复到上次位置。完成答题进入结果页后，自动清除题号 `ci` 记录，确保下次答题测试从头开始。
### 结果页

新增 **"保存本次结果"** 按钮。点击后返回首页的人格展示页面，会出现"查看保存记录"入口，方便随时回顾和对比上次测试的人格结果。
### 彩蛋页

点击页面底部的 **"不被定义，这只是参考"**，会触发隐藏彩蛋。彩蛋内容仅供参考——毕竟，**仅供参考，仅供参考，仅供参考**，重要的事情说三遍。


## 未来规划：想加但还没做的功能

### 双人相处报告——最具潜力的功能

访问 `/pair?a=INFJ&b=ENTP`，输入两个人的 MBTI 类型，直接生成"相处建议"。这是 MBTI 产品最自然的传播场景：**"来看看我们搭不搭"**。朋友圈里两个人各贴一张图，配一句"认证过了"——传播链条自己就完成了。

### 每日一签——让人愿意回来的理由

根据用户的 MBTI 类型，每天推送一句温和的"今日提醒"。比如 INFJ 看到的是：「你天生能听出别人没说出口的话——今天也记得听听自己的。」数据提前写成 JSON 文件即可，无需动态生成。

### 关系雷达图——一张图胜过千言万语

在结果页放一张极简雷达图，展示当前类型与另外 16 种类型的相处契合度。雷达图天然比文字列表更吸睛，也更容易被截图分享。

### 暗色模式（已完成）

内向型用户（尤其是 INxx）偏好深色背景阅读。Tailwind 的 `dark:` 前缀让实现成本很低，值得做。

### 其他小想法

- **2 秒开场动画**：进入测试时，四颗阵营色光点汇聚成 Logo
- **"回顶部"悬浮按钮**：结果页内容较长，右下角放一个快捷按钮

## 后记

做完这个项目，最大的感受就一个字：改。

从网站的整体主题框架，到卡片的点击动效，再到题目搜索精度、人格契合度与磨合度的内容——基本上能改的我都改了。做到后期，就是哪里看着不顺眼改哪里，实在改不动的先放一放，过两天再回头看，继续改。

所以总结下来一句话：**优化和 bug，永远改不完。**

> ps：说实话，做这些事情还真挺好玩的。

## 写在最后

MBTI 测出来的人格，并不代表你就是一个什么样的人，它更多只是一个帮助你认识自己的参考工具。如果你暂时还没办法对自己形成清晰的认知，不妨通过这类测试，先从一个角度开始了解自己。

当然，无论测试结果是什么，它都不能决定你往后的人生。人生充满不确定性，你会遇见很多人，经历很多事，而这些经历也会不断塑造你，让你成为一个与今天不同的自己。未来未知，但也正因为未知，才值得期待。

所以，不必因为一次测试的结果就轻易给自己下定义。保持客观地看待自己，接纳变化，也许这才是真正认识自己的开始。
